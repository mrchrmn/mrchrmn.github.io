Conslog

A VS Code extension for a better console.log() experience

 Repository      Marketplace

Conslog is a great time saver for JavaScript learner, making it super easy to add or remove console.logs to and from selected code.

The idea behind it was to make it easier to log the test cases that are sometimes provided with coding problems, eg. on Codewars or Leetcode, for example like this:

mySolution(testCase); // => true
mySolution("testCase"); // => "Casetest"

For a full description visit the Visual Studio Marketplace or the project’s Github page.

After getting to grips with how VS Code snippets and extensions work, the project was basically an exercise in regular expressions. Check out a few of the key features below to find out what regular expression is at work.

Log selection

It wraps the selection in console.logs. But only the part of the selection until the first semicolon is getting wrapped, the rest remains as is.

If multiple lines are selected, each line is treated like an individual selection, adding multiple console.logs. Skips empty lines and comment lines.

Behind the scenes

This is the regular expression that does the magic:

/^(?!\/\/)([^\r\n][^;\r\n]*)(;)?(.*)$/gm

Undo logs

Removes console.log from selected lines, unwrapping the argument. Reverts the effect of “Log selection” or “Log lines …”.

Behind the scenes

This is the regular expression that does the trick:

/console\.log\(|(?<=^ *console\.log\(.*)\)(?=;)/gm

Delete console logs

If it turns out that you really like console.log debugging, chances are that your code has plenty of lines temporarily logging variables. Get rid of them in one go with this snippet.

Behind the scenes

This is the regular expression that does the work:

/^ *console\.log\(.*(\r\n|\r|\n)?/gm
